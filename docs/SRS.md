**1. Requerimientos Funcionales (RF)**
   
RF01: El sistema deberá permitir el registro de usuarios mediante credenciales únicas.

RF02: El sistema deberá autenticar el acceso mediante validación de identidad.

RF03: El sistema deberá segregar interfaces y funcionalidades según el rol (Cliente/Emprendedor).

RF04: El sistema deberá desplegar un catálogo dinámico de emprendimientos en el módulo principal.

RF05: El sistema deberá proveer herramientas de búsqueda y filtrado por categorías.

RF06: El sistema deberá permitir el acceso al detalle técnico y comercial de cada producto.

RF07: El sistema deberá gestionar una persistencia de emprendimientos marcados como favoritos.

RF08: El sistema deberá administrar un carrito de compras con funciones de edición en tiempo real.

RF09: El sistema deberá procesar órdenes de pedido y gestionar métodos de pago.

RF10: El sistema deberá generar un historial transaccional de pedidos realizados.

RF11: El sistema deberá proporcionar formularios dinámicos para el registro de nuevos negocios.

RF12: El sistema deberá permitir la gestión integral (CRUD) del catálogo del emprendedor.

RF13: El sistema deberá notificar y monitorear solicitudes de servicio en tiempo real.

RF14: El sistema deberá permitir la actualización del estado de las solicitudes (Aceptar/Rechazar).

RF15: El sistema deberá integrar un canal de comunicación directa sincrónica (Chat).

RF16: El sistema deberá habilitar un módulo de calificación cuantitativa mediante estrellas.

RF17: El sistema deberá permitir la publicación de reseñas cualitativas en perfiles públicos.

RF18: El sistema deberá facultar al emprendedor para emitir respuestas a las reseñas recibidas.

RF19: El sistema deberá permitir la administración de eventos institucionales.

RF20: El sistema deberá facilitar la edición y gestión del perfil de identidad del usuario.




**2. Requerimientos No Funcionales (RNF)**  

RNF01: El sistema deberá ser desarrollado bajo una arquitectura multiplataforma utilizando el framework Flutter.

RNF02: El sistema deberá garantizar la persistencia y sincronización de datos en tiempo real mediante los servicios de Firebase.

RNF03: El sistema deberá implementar una interfaz adaptable que soporte el Modo Oscuro de forma dinámica.

RNF04: El sistema deberá gestionar el estado de los datos de manera eficiente mediante el patrón de diseño Provider.

RNF05: El sistema deberá cumplir con estándares de accesibilidad visual, permitiendo el escalado dinámico de tipografía.

RNF06: La arquitectura de software deberá ser modular y desacoplada, siguiendo el patrón de separación de capas (Modelos, Proveedores y Vistas).

RNF07: El sistema deberá garantizar un tiempo de respuesta y carga de información no mayor a 2 segundos.

RNF08: El diseño de la experiencia de usuario (UX) deberá regirse estrictamente por los principios de Material Design.

RNF09: El sistema deberá ejecutar validaciones sintácticas y lógicas en todos los campos de entrada de datos.

RNF10: El sistema deberá asegurar la integridad de la sesión mediante el servicio de autenticación Firebase Auth.

RNF11: El sistema deberá implementar algoritmos de optimización de imágenes para reducir el consumo de ancho de banda.

RNF12: La navegación del sistema deberá estructurarse mediante una barra de navegación persistente para mejorar la usabilidad.

RNF13: La interfaz gráfica deberá alinearse con la identidad corporativa y paleta cromática oficial de la UIDE.

RNF14: El sistema deberá contar con mecanismos de tolerancia a fallos para gestionar la pérdida de conectividad a internet.

RNF15: El sistema deberá optimizar el uso de hilos de procesamiento para garantizar un bajo consumo de batería.

RNF16: El código fuente deberá estar documentado técnicamente bajo estándares que faciliten su mantenimiento futuro.

RNF17: El sistema deberá utilizar una librería de iconografía descriptiva coherente con las acciones del usuario.

RNF18: El sistema deberá garantizar la confidencialidad de la información personal según las políticas de privacidad vigentes.

RNF19: La infraestructura del backend deberá asegurar una alta disponibilidad mediante servicios en la nube.

RNF20: El sistema deberá gestionar el envío de notificaciones locales para alertar sobre eventos críticos y mensajes.
